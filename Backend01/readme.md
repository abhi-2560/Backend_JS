Task: Write a short explanation (5–8 lines) describing what happened from request → response.


When I triggered a request from the browser, the client first built an HTTP request containing the URL, method (like GET or POST), and request headers.

This request was sent over the network to the target server using the HTTP/HTTPS protocol.

The server received the request, processed it (often involving backend logic and database queries), and generated a response.

The response included status information, response headers, and the response body (such as HTML or JSON data).

This response was then sent back to the browser over the network.

Finally, the browser parsed the response and rendered the content on the screen (or used it in the application).
